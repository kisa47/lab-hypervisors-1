## Требования к выполнению работы

- Компьютер с операционной системой Debian и установленным гипервизором KVM
- Образ диска виртальной noble-server-cloudimg-amd64.img

Научиться запускать виртуальные машины в гипервизоре KVM с использованием Qemu с заданными параметрами через терминал.

## Теоретическая часть

Гипервизор (hypervisor) является ключевым компонентом облачных технологий и виртуализации, играющим важную роль в современном мире информационных технологий. Чтобы лучше понять историю появления гипервизора, важно рассмотреть этапы его эволюции и причины возникновения.

Гипервизор - програмное обеспечение или аппаратная среда обеспечивающий выполнение одной или нескольких виртуальных машин, а также создание и управление ими, на одном физическом устройстве (сервере). Основная задача гипервизора состоит в предоставлении и координации ресурсов виртуальным машинам таким образом, чтобы они могли функционировать параллельно и независимо (изолированно) друг от друга, сохраняя эффективность и надежность всей системы.

KVM (Kernel-based Virtual Machine) — программное решение для аппаратной виртуализации в операционных системах Linux и её дистрибутивах. Это модуль ядра Linux, который превращает ОС в гипервизор первого типа (Bare Metal) — то есть работающий сразу поверх железа без лишних уровней абстракции.

QEMU (Quick Emulator) — программа с открытым исходным кодом для эмуляции аппаратного обеспечения различных платформ. Позволяет запускать виртуальные машины с различными операционными системами на одном физическом устройстве.

## Проведение лабораторной работы

### Подготовка образа диска виртуальной машины

Задать пароль для учетной записи root командой, где <pass> - пароль:
```bash
virt-customize -a  ~/images/noble-server-cloudimg-amd64.img --root-password password:<pass>
```

### Старт виртуальной машины

запустить виртуальную машину со следующими параметрами:
- cpu host: Использует возможности процессора хоста для ускорения работы виртуальной машины.
- type=q35: Машинный тип q35 обеспечивает поддержку современного оборудования, совместимого с ACPI.
- accel=kvm: Включает ускорение KVM (Kernel-based Virtual Machine) для повышения производительности виртуализации.
- объем выделенной оперативной памяти 2Gb
- с отключенным графическим интерфесом (вывод идёт непосредственно в консоль)
- в режиме снимка (не изменяя исходный образ диска)
- с переадресацией сетевых соединений с порта 2222 хоста на 22 порт виртуальной машины

```bash
qemu-system-x86_64  \
  -cpu host -machine type=q35,accel=kvm -m 2048 \
  -nographic \
  -snapshot \
  -netdev id=net00,type=user,hostfwd=tcp::2222-:22 \
  -device virtio-net-pci,netdev=net00 \
  -drive if=virtio,format=qcow2,file=/home/b415/images/noble-server-cloudimg-amd64.img
```
### Настройка виртуальной машины

После старта ВМ необходимо дождаться её запуска (может занять несколько минут) необходимо войти в виртуальную машину с использованием имени пользователя root и заданным на этапе "Задание пароля пользователя root для виртуальной машины" пароля.

Получить сетевой интерфейс командой ```ip link```

Далее необходимо создать файл /etc/netplan/50-cloud-init.yaml следующего содержания, где enp0s2 интерфейс:
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s2: 
      dhcp4: true
      dhcp6: false
```
И применить конфигурацию командой ```netplan apply```. После применения конфигурации проверить доступ в интернет выполнив ```ping vshte.ru```

Установить следующее программное обеспечение командой:
```bash
apt-get update && apt-get install -y memtester
```

### Получение данных о ОС виртуальной машины.

После успешного входа необходимо получить следующие данные:

- версию ядра ОС и архитектуру виртуальной машины командой ```uname -a```
- Объем оперативной памяти ВМ командой ```free -h```
- модель процессора командой ```lscpu```
- получить древо процессов командой ```pstree -Aptl```
- список сетевых интерфесов (кроме lo) командой ```ip link```
- список pci устройств командой ```lspci```

### Полученние данных с ОС хоста

Не выходя из ВМ и не завершая её работу открыть другую вкладку в терминале, собрать в нём аналогичные данные и вписать их в раздел "Полученные результаты".

### Нагрузочное тестирование и поломка ВМ

Выполнить в Виртуальной машине команду для нагрузочного тестирования ОЗУ:
```bash
memtester $(free -m | awk '/Mem/{print $6}')M 1
```

Перейти в терминал хоста и пронаблюдать как изменилось потребление оперативной памяти. Записать результат команды во время нагрузочного тестирования ```free -h``` в раздел "Полученные результаты".

После завершения тестирование, необходимо выполнить умышленную полому виртуальной машины. Для это попробуем начать полную перезапись данных диска ВМ.

Получим устройство которое нужно переписать нулями командой:
```bash
df / | tail -n 1 | awk '{print $1}'
```

Само устройсто не будет иметь дисковой метки тома (цифры), например вывод команды */dev/vda1*, значит нужное нам устройство */dev/vda*.

И выполним запись всего диска "нулями":
```bash
dd if=/dev/zero of=/dev/vda bs=1M status=progress
```

После завершения попробовать разные команды. Посмотреть как система работает.

Переключиться в терминал хоста и посмотреть, не затронули действия по уничтожению данных на работающей ВМ работсопособность узла.

Записать полученные наблюдения в раздеое "Полученные результаты"
